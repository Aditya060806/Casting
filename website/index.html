Features

Everything you need, nothing you don't.

Casting handles the boilerplate so you can focus on the interesting parts.

🪄

Zero-config auto-detect

Reads package.json and tsconfig.json to infer TypeScript, styling, test library, and framework.

⚡

One-command scaffolding

casting c Box creates the folder, component, style, test, and barrel file at once.

🧩

Three generators

Components, custom hooks, and contexts with a Provider and a typed consumer hook.

🎨

Format on generate

Output is piped through your project's Prettier config so it looks hand-written.

🔍

Dry-run diff preview

See a colored, line-by-line preview of every file before anything touches disk.

🎛️

Interactive mode

Run casting with no arguments for an arrow-key menu.

🧠

Typed config

casting.json ships a JSON schema for editor autocomplete and validation.

🧵

Next.js aware

Adds the 'use client' directive to contexts automatically when it detects Next.js.

        How it works
        
A small, predictable pipeline.

        
Every run follows the same six steps — from your command to formatted files on disk.

      


      
    
        
  1. 1
    Locate
    Confirms you're at the project root.
    2. 
        
  2. 2
    Configure
    Loads casting.json, or auto-detects your stack on first run.
    3. 
        
  3. 3
    Resolve
    Picks built-in templates or your custom overrides.
    4. 
        
  4. 4
    Transform
    Swaps the TemplateName keyword into 7 casings.
    5. 
        
  5. 5
    Format
    Runs the result through your Prettier config.
    6. 
        
  6. 6
    Emit
    Writes files (never overwrites) and prints a summary.
    7.

        Documentation
        
Get productive in a minute.

        
Install, generate, and configure. Everything you need is right here.

      


      

        

          
          
          
          
          
          
        


        

          

            
Global install (the executable is casting):

            
npm i -g casting-cli
casting component Button

            
Or run it on demand with npx:

            
npx casting-cli init --auto
npx casting-cli component Box

            
Requires Node.js 22+ and npm 10+.

          


          

            
Generate one — or several — components at once:

            
casting component Box
casting component Box Card Modal

            
import type { FC } from 'react';
import styles from './Box.module.css';

interface BoxProps {}

const Box: FC<BoxProps> = () => (
  <div className={styles.Box} data-testid="Box">
    Box Component
  </div>
);

export default Box;

          


          

            
Scaffold a custom hook and its test:

            
casting hook useToggle

            
import { useCallback, useState } from 'react';

export function useToggle() {
  const [value, setValue] = useState<unknown>(null);
  const reset = useCallback(() => setValue(null), []);
  return { value, setValue, reset } as const;
}

          


          

            
Generate a context with a Provider and a typed consumer hook:

            
casting context Theme
casting context Theme --client   # prepends 'use client'

            
export function useTheme() {
  const context = useContext(ThemeContext);
  if (context === undefined) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;
}

          


          

            
Create the config by auto-detecting your stack:

            
casting init --auto

            
{
  "usesTypeScript": true,
  "usesCssModule": true,
  "cssPreprocessor": "scss",
  "testLibrary": "Testing Library",
  "component": {
    "default": {
      "path": "src/components",
      "withStyle": true,
      "withTest": true,
      "withIndex": false
    }
  }
}

          


          

            

              
--dry-runPreview with a colored diff, write nothing

              
-f, --flatGenerate files directly in path (no folder)

              
-y, --yesSkip prompts, use detected defaults

              
-p, --pathOverride the output directory

              
--typeUse a custom component type from your config

              
--withIndexGenerate a barrel index file

              
--clientAdd the 'use client' directive (contexts)

              
--customDirectoryOverride the generated folder name
